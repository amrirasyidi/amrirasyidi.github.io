## Enabling Google Map API

Google provides an API specifically to fetch imagery, it is covered under
the "Maps Static API", which you can enable by following the guide in this
<a href="https://developers.google.com/maps/documentation/maps-static/get-api-key?setupProd=enable#console" target="_blank" rel="noreferrer noopener">page</a>.
If you're just trying things out, you shouldn't worry too much about the
pricing, as long as your "trying" don't exceed 10,000 API calls (if I
correctly intepreted this
<a href="https://developers.google.com/maps/billing-and-pricing/pricing" target="_blank" rel="noreferrer noopener">page</a>).

Once you click "Enable the Maps Static API", you will need to provide some
information, mostly regarding the billing, then in the end, you will be
shown the API key. This API key will be needed to fetch the image later.

## Fetching the data

To programmatically fetch the satellite imagery using Python, we can use
`request`. On the documentation, the request url would look like this:

```
https://maps.googleapis.com/maps/api/staticmap?center=40.714728,-73.998672&zoom=12&size=400x400&maptype=satellite&key=YOUR_API_KEY&signature=YOUR_SIGNATURE
```

To use it in request, we could use something like this

```python
import requests

base_url = "https://maps.googleapis.com/maps/api/staticmap"

# Build API request parameters
params = {
    'center': f"{latitude},{longitude}",
    'zoom': Config.ZOOM_LEVEL,
    'size': Config.IMAGE_SIZE,
    'maptype': Config.MAP_TYPE,
    'format': Config.IMAGE_FORMAT,
    'key': Config.api_key
}

response = requests.get(base_url, params=params, timeout=30)
```

In the case above, `Config` is a python class which contains the
relevant parameters to be fed in the request. On top of that, you also need
the value for `latitude` and `longitude`.

The response is not yet an image, to make it an image, you will need to read
the content of the response, in this case, we can use `PIL.Image`
library.

```python
from PIL import Image

if response.status_code == 200:
    # Verify the response is actually an image
    try:
        image = Image.open(BytesIO(response.content))

        # Save the image
        with open(filepath, 'wb') as f:
            f.write(response.content)

    except Exception as e:
        logger.error(f"Invalid image data received: {e}")
        return None
```

Then that's it, there's only 2 "steps" to get the satellite imageries from
google map, that is to enable the api and then actually fetch the data. Once
you got the required data, you can start labelling the images to train our
custom model, which will be covered in the next part.
